# Debugging

- devtools 에서 확인할 수 없는 오류는 실기기에서 테스트해야 한다.
- css 애니메이션, 자바스크립트 동작 테스트 시 유용하다.

## 로컬 환경에서 모바일 기기 테스트

- 같은 공유기를 사용할 경우에만 확인 가능하다.
- ip주소:포트주소/index.html 에서, 테스트할 기기가 연결된 주소로 ip주소를 변경한다.

## Android 기기 디버깅

- 화면 미러링 가능
- 인앱브라우저도 테스트 가능

1. 안드로이드 기기에 개발자 옵션 활성화
   - 기기 설정에서 빌드 번호를 빠르게 7번 터치
2. 기기의 개발자 옵션에서 USB 디버깅 활성화
3. 안드로이드 기기와 랩탑 USB 연결
4. 랩탑 크롬의 주소창에 chrome://inspect 입력

## iOS/iPadOS 기기 디버깅

- Mac 에서만 무료로 가능. Windows 에서는 inspect.dev 에서 유료
- 화면 미러링 불가

1. 사파리 설정에서 개발자 기능 보기 설정
2. 아이폰 - 설정 - safari - 고급 - 웹 인스펙터
3. Mac 기기와 아이폰 연결
   - 팝업창에서 신뢰 선택
4. Mac 사파리 메뉴 - '개발자용' 연결된 기기 확인
